🔵Component Communication:- Exchange the data between in any 2 component
➡️A. Parent to Child → @Input()
whenever we send the data from parent to child.

// parent HTML
<app-child [name]='ravi'></app-child>

// child HTML
<h5>{{name}}</h5>

// child TS
@Input() name = ' ';


➡️B. Child to Parent → @Output() with EventEmitter

// child HTML
<button (click)="saveUser()"></button>

// child TS
@Output sendData = new EventEmitter<string>();

let name = 'mahendra';
saveUser(){
    this.sendData.emit(this.name);
}

// Parent HTML
<app-child (sendData)="getParentData($event)"> </app-child>
<h5>{{recievedData}}</h5>

// Parent TS
let recievedData:any=null;
getParentData(data?:string){
    this.recievedData = data;
}

➡️@ViewChild() (Call Child Method from Parent)
Used when parent want to access child properties/methods.
Instead of child sending data to parent, @ViewChild() lets the parent directly talk to the child.

// child TS
let message = 'hello from chid component';

showMessage(){
    return this.message;
}

// parent HTML
<app-child></app-child>

// Parent TS
import {childComponent} from './child.component';

@ViewChild(childComponent) child! : childComponent;

ngAfterViewInit(){
    console.log(this.child.showMessage());
}

🔵What is the Angular lifecycle?
Angular lifecycle is the series of stages a component passes through, along with lifecycle hook methods that run at each stage
Main lifecycle phases
1. Creation phase
ngOnChanges() → runs when input data changes
ngOnInit() → runs once after component is created
2. Change Detection phase
ngDoCheck() → runs on every change detection
3. Rendering phase
ngAfterContentInit() → after content is loaded
ngAfterContentChecked() → after content is checked
ngAfterViewInit() → after view (HTML) is loaded
ngAfterViewChecked() → after view is checked
4. Destruction phase
ngOnDestroy() → runs before component is destroyed


➡️ngOnInit():- after component initialize ngOnInit will run.

➡️ngAfterContnetInit():
whenever we send any html content data from parent compoennt to child component --> Content Projection

<app-info-panel>
  <p #projectedText>

  </p>
</app-info-panel>

#projectedText - template reference variable of DOM


child.comp.html
<div>
  <h3>Child Component</h3>
  <ng-content></ng-content>
</div>

child.comp.ts
@ContentChild('projectedText') projectedText!: ElementRef; 

ContentChild -- searching the reference variable in projected DOM
ng-content -- render the data in html

ngAfterContentInit(): void {

➡️ What is ng-container?
ng-container means invisible wrapper 

<h *ngIf="logIn">Welcome </h>
<p *ngIf="logIn"> user log in </p

<div *ngIf="logIn">
<h>Welcome </h>
<p> user log in </p
</div

<ng-container *ngIf="logIn">
<h>Welcome </h>
<p> user log in </p
</ng-container>

div, p, h1 tags will be show in DOM but ng-container is not show in DOM 
if we are using ng-container then not need to use extra div
