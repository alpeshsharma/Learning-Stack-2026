21. Why do we use Observables in Angular applications?
Observable gives data → subscribe receives data

>> Observable
it is a stream that emits data over time when an async operation (like API call) completes.

>> What is subscribe()?
Subscribe is a method used to execute an Observable and receive the emitted data when the asynchronous operation
(like an API call) completes.
Subscribe is used to get data from an Observable when the API response arrives.

>> Why do we need to subscribe?
In RxJS, Observables are lazy. So without subscribe(), the Observable will not execute,
and no API call or data emission will happen.

>> What are the handlers inside subscribe()?
subscribe() can take three handlers:

next → for data
error → for error handling
complete → when stream finishes

25. Can you explain Angular Guards?
Angular Guards are used to secure protect and control routes/pages in an Angular application.
They check conditions before opening, leaving, or loading a page.

For example:
If a user is not logged in, guards can stop access to the dashboard page.

1. CanActivate
CanActivate guard is used to check whether a user can access/enter a route or not.

Example:
If user is not logged in, dashboard page will not open.

2. CanDeactivate
CanDeactivate guard is used to check whether a user can leave a page or not.

Example:
If user is filling a form and clicks another page,
a popup message appears

3. CanActivateChild
CanActivateChild guard is used to protect all child routes of a parent route.

Instead of applying guard on every page,
we apply one guard on parent route.

4. CanMatch
CanMatch guard is used to decide whether a route should load/match or not.

Only admin user can open admin panel.

If normal user tries:
route will not load.

CanActivate      → Enter page?
CanDeactivate    → Leave page?
CanActivateChild → Protect child pages?
CanMatch         → Load/match route?

==> HTTP Interceptor is a common place where Angular catches every API request/response before it reaches your component.
==> HTTP Interceptor is used to intercept all HTTP requests and responses globally for common tasks like authentication, error handling, and logging.
Flow:
Component → Interceptor → API
API → Interceptor → Component

• Handle common errors (401, 404, 500)

29. Can you explain Server-Side Rendering (SSR) and Client-Side Rendering (CSR)?
Server-Side Rendering (SSR) is a rendering method where the server generates the complete HTML page on the server side and sends the ready page to the browser.
Client-Side Rendering (CSR) is a rendering method where the browser receives basic HTML and JavaScript, then fetches data and builds the page on the client side.

The main difference between SSR and CSR is where the page is rendered.
In SSR, rendering happens on the server, while in CSR, rendering happens in the browser (client)
