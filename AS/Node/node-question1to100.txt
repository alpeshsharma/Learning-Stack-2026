========================
NODE.JS INTERVIEW ROADMAP
(LOW → HIGH LEVEL ORDER)
========================

LEVEL 0 — FOUNDATION

1. Node.js kya hai?
Ans:
Node.js JavaScript runtime environment hai jo browser ke bahar JavaScript run karne deta hai.

2.Node browser ke bahar JS kaise chalata hai?

V8 engine open-source hai. Node.js us engine ko embed karta hai, isliye browser ke bina bhi JavaScript execute ho jati hai.
Browser APIs ki jagah Node backend APIs provide karta hai.

CONFUSION:
“Agar V8 browser ke andar hota hai, toh Node browser ke bahar JavaScript kaise chalata hai?”

ANSWER:

V8 sirf browser ka part nahi hai.
V8 ek separate open-source JavaScript engine hai.

Chrome browser ke andar V8 laga hota hai.

Chrome Structure:

Chrome Browser
--------------------------------
UI
Tabs
Address Bar
DOM
Window Object
+
V8 Engine
--------------------------------

Browser me jab hum JavaScript likhte hain:

console.log("hello")

toh Chrome ke andar laga V8 engine us code ko execute karta hai.

--------------------------------------------

Node.js ne kya kiya?

Node ne poora browser use nahi kiya.

Node ne sirf V8 engine ko uthaya (embed kiya)
aur apna alag runtime bana diya.

Node Structure:

Node.js Runtime
--------------------------------
V8 Engine
+
File System (fs)
HTTP Module
OS Access
Process APIs
Event Loop
--------------------------------

Jab tum terminal me likhte ho:

node app.js

Tab actual flow hota hai:

JavaScript Code
       ↓
Node Runtime (node.exe)
       ↓
Embedded V8 Engine
       ↓
Machine Code
       ↓
CPU execute karta hai

Matlab:

Browser me:
JS → Chrome → V8 → Execute

Node me:
JS → Node Runtime → V8 → Execute

DIFFERENCE:

Browser APIs:
window
document
localStorage
DOM

Node APIs:
fs
http
path
process

Browser me ye chalega:
document.getElementById()

Node me error dega.

Node me ye chalega:
const fs = require("fs")

Browser me error dega.

MOST IMPORTANT LINE (YAAD KARLO):

JavaScript ko chalane ke liye engine chahiye.
Browser mandatory nahi hai.

Browser ek jagah hai jahan V8 laga hota hai.
Node dusri jagah hai jahan wahi V8 Node runtime ke andar embed kiya gaya hai.

INTERVIEW ANSWER:

“V8 engine open-source hai. Node.js ne us engine ko apne runtime (node executable) ke andar embed kiya hua hai.
 Isliye browser ke bina bhi JavaScript execute ho jati hai. Browser APIs ki jagah Node backend APIs provide karta hai.”

3. Node.js kyun bana?
Ans:
Browser ke bahar JavaScript chalane aur scalable backend banane ke liye.

4. Node.js ki khas baat?
Ans:
Fast, non-blocking, event-driven aur scalable.

5. Node.js single-threaded hai?
Ans:
Haan, JavaScript execution single thread pe hota hai.

Follow-up:
Internally libuv thread pool heavy tasks handle karta hai.

6. Node.js synchronous hai ya asynchronous?
Ans:
Dono support karta hai, but mostly asynchronous architecture use hota hai.

7. Blocking vs Non-blocking?
Ans:
Blocking → code wait karta hai.
Non-blocking → next task continue hota hai.

Example:
readFileSync() → blocking
readFile() → non-blocking

8. Event-driven architecture kya hota hai?
Ans:
Events trigger hote hain aur unke basis pe code execute hota hai.

9. Event loop kya hai?
Ans:
Call stack aur callback queue ko manage karta hai aur async tasks execute karata hai.

10. Node.js fast kyun hai?
Ans:
V8 engine aur non-blocking architecture ki wajah se.
